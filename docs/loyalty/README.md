# Система лояльности — рабочая зона

Материалы по функционалу лояльности: бизнес-логика, UI (пользователь и админ), техспека.

## Файлы

| Файл | Назначение |
|------|------------|
| [business-logic.md](./business-logic.md) | XP, уровни, только % скидки, стекинг с промо, миграции, freeze при `LOYALTY_ENABLED=false` |
| [ui-user-admin.md](./ui-user-admin.md) | Сценарии экранов, кнопки; **вход из «Мой VPN»**, не из главного меню |

## Статус процесса (doc-coauthoring)

- Этап **сбора контекста** закрыт ответами по правилам XP, стекингу, freeze, миграции, бонусу %, HWID/тарифам.
- Этап **уточнения структуры**: бизнес-док и UI-док обновлены; пользовательский **флоу «Мой VPN»** нужно финализировать перед кодом (кнопка vs блок+«Подробнее»).
- Ретро-бэкофилл: для старых пользователей — **сумма потраченных ₽** (жизненная модель XP); резервные правила по строке — см. `business-logic.md`.

## Связь с проектом

- `SALES_MODE`: `classic` | `tariffs`.
- Промокоды: pending % — стекинг с лояльностью **аддитивный**, потолок **`LOYALTY_MAX_TOTAL_DISCOUNT_PERCENT`** (см. `business-logic.md`, `technical.md`).
- После удаления **Tribute** особых веток «без скидки» для этого провайдера не планируется.
